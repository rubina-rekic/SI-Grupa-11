# Sprint Goal – Sprint 3

## Sprint broj
3

## Sprint cilj
Uspostaviti analitičke i arhitektonske temelje sistema kroz izradu Risk Registera,
Domain/Use Case modela, Architecture Overviewa i Test Strategije, kako bi tim
imao jasnu tehničku sliku sistema prije početka implementacije.

## Ključne stavke koje tim želi završiti
- [ ] Risk Register – identifikacija i analiza ključnih projektnih rizika
- [ ] Domain Model i/ili Use Case Model – modeliranje domene i glavnih slučajeva korištenja
- [ ] Architecture Overview – definisanje arhitektonskog pravca sistema
- [ ] Test Strategy – definisanje pristupa testiranju i nivoa testiranja

## Rizici i zavisnosti
- Arhitektonske odluke moraju biti utemeljene na zahtjevima iz Sprinta 2,
  a ne na nasumičnom tehnološkom izboru
- Domain model zavisi od user stories i acceptance criteria definisanih u prethodnom sprintu
- Nejasnoće oko domene treba razjasniti s Product Ownerom prije finalizacije modela
